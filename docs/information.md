# О проекте
Этот проект предназначен для упрощения поиска и отбора релевантных вакансий с портала HH.ru и составления сопроводительных писем работодателям
# Пользовательский пайплайн
## Регистрация
Пользователь регистрируется на платформе с помощью email и пароля, после чего подтверждает свой email
## Связь с HH
Для работы с продуктом пользователю необходимо привязать 1 или несколько своих аккаунтов HeadHunter. Для этого после регистрации UX предлагает пользователю авторизоваться с помощью HH
# Парсинг резюме
После авторизации в аккаунте HeadHunter пользователь должен выбрать резюме, на основании которого будет производиться поиск вакансий. После выбора парсер в фоновом режиме спарсит все доступные резюме. Пользователь сможет с ними ознакомиться
В любой момент пользователь может перепарсить вакансии
# AI оценка релевантности
После парсинга резюме пользователь может воспользоваться функцией AI оценки релевантности. Модель анализирует резюме с HH или загруженное пользователем, после чего выставляет оценку от 1 до 100. Пользователь сможет отсортировать резюме по этой оценке и начать отклик с тех, что более релевантны для него
# AI составление сопроводительного письма
Пользователь может сгенерировать сопроводительное письмо под конкретную вакансию
# Статус вакансии
Пользователь может либо убрать эту вакансию и она ему никогда больше не попадётся, либо пользователь может пометить вручную, что откликнулся на неё
# Архитектура
## Main Frontend
**Техническое наименование:** "main-fe"
### Зоны ответственности:
- UX
- UI
- Доставка уведомлений пользователю
### Технологии
- NextJS - основной фреймворк
### Связи
- HTTP - `main-be`
- SSE - `main-be`
## Main Backend
**Техническое наименование:** "main-be"
### Зоны ответственности:
- Регистрация пользователей
- Авторизация пользователей
- Аутентификация пользователей
- Маршрутизация трафика
- Управление пользователями
- Уведомления
### Технологии
- FastAPI - Backend
- PostgreSQL - база данных
- SSE - уведомления на `main-fe`
### Связи
- HTTP - `main-fe` (управление сервисом)
- SSE - `main-fe` (доставка уведомлений)
- HTTP - `profile-service` (авторизация, управление профилями)
- HTTP - `vacancy-service` (передача вакансий, управление статусом вакансий)
- HTTP - `ai-service` (настройка AI)
- HTTP - `notification-service` (история уведомлений, настройка)
- NATS JetStream - `notification-service` (передача уведомлений в UI пользователя)
## HH profile service
**Техническое наименование:** "profile-service"
### Зоны ответственности:
- Хранение связей пользователей HH с пользователями системы
- Хранение access и refresh токенов HH
- Шифрование и дешифрование токенов HH
- OAuth авторизация пользователя
- Передача резюме пользователя, на которую ищутся вакансии
### Технологии
- FastAPI - Backend
- PostgreSQL - база данных
- aiohttp - авторизация в HH
- cryptography - шифрование/дешифровка токенов
### Связи
- HTTP - `main-be` (авторизация, управление профилями)
- HTTP - `vacancy-service` (выдача токена)
## HH vacancy service
**Техническое наименование:** "vacancy-service"
### Зоны ответственности:
- Сбор вакансий пользователя
- Хранение вакансий и статусов этих вакансий по отношению к пользователю
- Вывод вакансий
- Управление AI сервисом для процента релевантности
- Управление AI сервисом для генерации сопроводительных писем
### Технологии
- FastAPI - Backend
- PostgreSQL - база данных
- aiohttp - парсинг вакансий
- celery - очередь на парсинг вакансий
### Связи
- HTTP - `main-be` (передача вакансий, управление статусом вакансий, управление парсингом вакансий)
- HTTP - `profile-service` (получение токена)
- NATS JetStream - `ai-service` (постановка задач в очередь, получение обновлений)
- NATS JetStream - `notification-service` (отправка уведомлений пользователю)
## AI Service
**Техническое наименование:** "ai-service"
### Зоны ответственности:
- Выявление процента релевантности вакансии по отношению к резюме
- Генерация сопроводительных писем к вакансии
- Управление настройками AI (модели, промпты)
- Хранение статистики потребления токенов
### Технологии
- FastAPI - Backend
- PostgreSQL - база данных
- openAI - взаимодействие с OpenRouter
- celery - очередь на AI запросы
### Связи
- HTTP - `main-be` (настройка AI)
- NATS JetStream - `vacancy-service` (приём задач в очередь, передача обновлений)
## Notification Service
**Техническое наименование:** "notification-service"
### Зоны ответственности:
- Доставка уведомлений о событиях
- Доставка технических уведомлений
### Технологии
- FastAPI - Backend
- PostgreSQL - база данных (хранение уведомлений и настроек пользователя)
- celery - очередь на доставку уведомлений
### Связи
- HTTP - `main-be` (история уведомлений, настройка)
- NATS JetStream - `main-be` (передача уведомлений в UI пользователя)
