# Контекст
Мы начинаем работу над `vacancy_service`. Первым нашим шагом будет система хранения и обновления общедоступных справочников HeadHunter
# Какие справочники нужны
Для периодической синхронизации доступны:
- GET /dictionaries;
- GET /areas, /areas/{area_id}, /areas/countries;
- GET /professional_roles;
- GET /industries;
- GET /metro, /metro/{city_id};
- GET /languages.
# Что нужно сделать
Каждую неделю нам нужно выполнять запросы к HH и заполнять все справочники в нашу БД
В БД на каждый справочник нужно создать таблицу, которая будет хранить дамп этих справочников. Каждый справочник храним не как JSON, а как запись в таблице
# Автоматизация
Нам нужно обновлять значения всех справочников либо раз в неделю (или любое другое количество часов, которое должно задаваться в `.env`), либо если эти справочники отсутствуют. Для этого давай будем использовать Celery и CeleryBeat. Брокер - общий redis из инфраструктуры. Celery и Beat - отдельные на каждый микросервис
Beat каждый час проверяет актуальность справочников и сравнивает количество часов после последней синхронизации
Я предлагаю вести отдельную таблицу со справочниками и статусом их обновления. Никакие методы для этого пока что не нужны. Может быть позже будут нужны
# Main Backend
Нужны будут методы, которые позволят из сервиса получать все справочники по LIST и GET методам. POST, PATCH и DELETE не нужны - этими записями управляет исключительно автоматизация
Потом значения этих справочников будут подставляться в форму фильтров
Разумеется, нужен хотя бы минимальный фильтр по тексту (регистронезависимое регулярное выражение)
# Принципы синхронизации
HH предупреждает, что значения справочников могут меняться. Поэтому их лучше синхронизировать через upsert и хранить active, а не считать названия и состав неизменными.
Дедуплицировать по ID справочника, который предоставляет HeadHunter. Если при запросе справочников выясняется, что какие-то элементы отсутствуют, то нужно снять статус active
# Тестирование
В качестве e2e теста необходимо проверить следующие сценарии:
- справочники отсутствуют -> полная синхронизация справочников
- некоторые справочники отсутствуют -> синхронизация отсутствующих справочников
- некоторые справочники обновлялись позже указанного количества часов -> обновление справочников
- все справочники актуальны -> ничего не делать
