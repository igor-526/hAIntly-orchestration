# Контекст
Сейчас для регистрации и авторизации используются 2 поля - логин и email
Нам нужно полностью убрать login в качестве уникального идентификатора и оставить только email в качестве логина
# Задача
- Убери из модели пользователя лишнее поле
- Доработай на основном backend регистрацию, аутентификацию  и авторизацию
- Сделай миграцию и примени её методом перезапуска контейнера миграции
- Перезапусти приложение
- Выполни smoke тестирование API. Зарегистрируй нового пользователя, авторизуйся и аутентифицируйся под ним
- Доработай страницу `/login` на основном frontend под нововведения
