# Контекст
Мы продолжаем работать над сервисом вакансий
В задачах 07 и 08 были реализованы системы хранения справочников и хранения/управления фильтрами. В этой итерации мы будем реализовывать систему выдачи вакансий
# Доступ
Выбранный профиль HeadHunter хранится в Main Backend
Токены в зашифрованном виде хранятся в Profile Service
Для получения списка ваканский нам необходимо обратиться к MainBackend за выбранным аккаунтом и потом к Profile Service за access токеном этого аккаунта
# Передача токена
Profile Service хранит токен в зашифрованном виде. Нам нужно реализовать сервисный GET метод, который будет возвращать актуальный access токен уже в незашифрованном виде.
