# Контекст
Мы начинаем реализацию сервиса AI
Его движком будет OpenRouter, я положил его токен в файл `.env`
Сервис будет решать следующие задачи:
- генерация фильтров по резюме
- оценка релевантности резюме
- генерация сопроводительного письма
В этой задаче мы будем реализовывать только движок этого сервиса
# Задачи
## Заложить архитектуру под расширение действий
Есть вероятность, что этими действиями функционал не будет ограничен. При разработке необходимо учитывать то, что их может быть больше в будущем.
## Уведомления пользователей
Сервис notification ещё не реализован. Нужно заранее заложить то, что после выполнения действия ИИшкой нужно будет об этом уведомить пользователя
Схема будет следующей: сервис будет публиковать в NATS событие, которое произошло. Например, генерация фильтров закончена. Сервис уведомлений будет подписан на канал ai и по этому действию будет дальше работать с уведомлениями
## Реализовать клиент OpenRouter и структуру БД
Идея следующая: пользователи на каждое действие смогут настраивать модели и промпты под свои нужды
Для этого нужно заложить в БД хорошую архитектуру, чтобы пользователи могли выбирать из БД доступные модели и редактировать промпт
По промптам идея следующая - есть default промпты, которые мы засидируем, а есть пользовательские промпты, которые пользователи смогут сохранять в неограниченных количествах.
Для сидов возьми 3-5 самых популярных моделей из документации https://openrouter.ai/models
Для промптов сделай заготовку под действие "генерация фильтров по резюме"
## Статистика
Нам необходимо отслеживать каждое действие с нейронками. Создай в БД таблицу, которая будет хранить все параметры, которые доступны при взаимодействии и нейросетью
Потом эти данные будут нужны для подсчёта расходов, использования, популярности модели и тд. В общем, храним всё, что возможно
# Взаимодействие между микросервисами
Я предполагаю следующую схему. Так как у сервиса будет определённый набор действий, то сервис будет принимать запрос по типу
```json
{
	"action": "generate_filters",
	"user": "...",
	"data": "..."
}
```
Эту схему мы будем потом продумывать под каждое действие. Количество необходимых полей может варьироваться в зависимости от действия. Нужен будет приоритет, так как некоторые задачи будут выполняться в фоне
Нужна будет настройка максимального количества асинхронных задач
Учти, что этот микросервис с высокой долей вероятности будет скейлиться
