## MODIFIED Requirements

### Requirement: Страница входа и регистрации
`main-fe` MUST предоставлять маршрут `/login` с текстовым логотипом «HAIntly», полноширинным эксклюзивным MUI `ToggleButtonGroup` режимов «Вход» и «Регистрация», формой текущего режима и соответствующей кнопкой отправки.

#### Scenario: Первое открытие страницы входа
- **WHEN** неаутентифицированный пользователь открывает `/login`
- **THEN** система показывает выбранный MUI Toggle Button «Вход» с полями email и пароля и кнопкой «Войти»

#### Scenario: Переключение на регистрацию
- **WHEN** пользователь выбирает Toggle Button «Регистрация»
- **THEN** система показывает поля email и пароля и кнопку «Зарегистрироваться», а группа Toggle Button остаётся на всю ширину формы с визуальным расстоянием между вариантами

#### Scenario: Выбранный режим нельзя снять
- **WHEN** пользователь повторно активирует уже выбранный Toggle Button
- **THEN** система сохраняет текущий режим формы и один из двух вариантов остаётся выбранным

#### Scenario: Сброс чувствительного состояния при переключении
- **WHEN** пользователь переключает режим формы
- **THEN** система очищает пароль, ошибки полей и общую ошибку предыдущей отправки

## ADDED Requirements

### Requirement: Пользователь завершает сессию через main-be
`main-fe` MUST отправлять выход только в `main-be` запросом `POST /api/auth/logout` с cookie credentials, MUST блокировать повторную отправку во время запроса и MUST переводить клиентское состояние в `anonymous` только после успешного ответа `204`.

#### Scenario: Успешный выход
- **WHEN** аутентифицированный пользователь выбирает «Выход» и `main-be` отвечает статусом `204`
- **THEN** система очищает пользователя из клиентского auth state и заменяет защищённый маршрут на `/login`

#### Scenario: Повторный выход во время запроса
- **WHEN** logout-запрос ещё выполняется
- **THEN** действие «Выход» показывает pending-состояние и не отправляет второй запрос

#### Scenario: Ошибка выхода
- **WHEN** logout-запрос завершается сетевой или серверной ошибкой
- **THEN** система сохраняет текущее authenticated state, показывает безопасную ошибку и позволяет повторить выход

#### Scenario: Безопасная сериализация выхода
- **WHEN** feature auth вызывает logout endpoint
- **THEN** API client использует настроенный base URL и cookie credentials без чтения, хранения, логирования или передачи auth token из браузера
