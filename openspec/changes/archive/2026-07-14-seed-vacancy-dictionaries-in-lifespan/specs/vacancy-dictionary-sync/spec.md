## MODIFIED Requirements

### Requirement: Реестр состояния синхронизации
Система MUST вести отдельное состояние для `dictionaries`, `areas`, `countries`, `professional_roles`, `industries`, `metro` и `languages`, включая UUID primary key `id`, обязательное уникальное логическое `name`, время последней попытки, время последней успешной полной синхронизации, текущий результат и безопасное описание ошибки. Каждому из семи logical names MUST соответствовать явный фиксированный UUID из единого seed definition. Поиск и upsert фоновой/операционной синхронизации MUST выполняться по уникальному `name`, не используя UUID как логическое имя снимка и не заменяя его primary key.

#### Scenario: Успешная синхронизация
- **WHEN** снимок справочника полностью применён и DB-транзакция зафиксирована
- **THEN** состояние этого справочника сохраняет фиксированный UUID, отмечается успешным и `last_success_at` получает время завершения

#### Scenario: Ошибка до commit
- **WHEN** загрузка, валидация или применение снимка завершается ошибкой
- **THEN** ошибка фиксируется для строки с тем же уникальным `name`, а её фиксированный UUID и прежний `last_success_at` не изменяются

#### Scenario: Повторное обеспечение состояния
- **WHEN** lifespan, scheduler, worker или operational seed повторно обеспечивает наличие состояния для существующего логического имени
- **THEN** новая строка не создаётся и существующая каноническая пара UUID/name сохраняется

#### Scenario: Backfill существующих состояний
- **WHEN** migration/backfill встречает ранее созданную строку канонического name со случайным UUID
- **THEN** он переводит строку на зафиксированный UUID с сохранением status, timestamps и безопасного error, предварительно проверив collisions и согласованно обновив существующие ссылки, либо атомарно завершается ошибкой

#### Scenario: UUID conflict при backfill
- **WHEN** целевой фиксированный UUID уже принадлежит несовместимой строке или данные нельзя согласованно обновить
- **THEN** migration/backfill завершается ошибкой без частичного commit и не оставляет неоднозначное отображение logical name на UUID
