## Purpose

Определить UI рабочей области поиска вакансий в `main-fe` — список слева, детали справа, интеграция с Drawer фильтров.

## ADDED Requirements

### Requirement: Список вакансий в левой панели
В левой панели `WorkspaceShell` MUST отображаться список вакансий с infinite scroll. Каждая карточка MUST содержать: наименование вакансии, наименование работодателя, уровень зарплаты (от/до/валюта), локацию (город), ключевые навыки (snippet). Начальный размер страницы — 30 вакансий.

#### Scenario: Загрузка списка при открытии
- **WHEN** пользователь открывает главную страницу с привязанным HH аккаунтом
- **THEN** автоматически выполняется поиск вакансий без фильтрации (`page=0, per_page=30`), результаты отображаются в левой панели

#### Scenario: Пустой результат поиска
- **WHEN** поиск не вернул вакансий
- **THEN** отображается сообщение «Вакансии не найдены»

#### Scenario: Ошибка загрузки
- **WHEN** запрос поиска завершился ошибкой
- **THEN** отображается сообщение об ошибке с возможностью повторить

#### Scenario: Карточка вакансии без зарплаты
- **WHEN** вакансия не содержит информации о зарплате
- **THEN** блок зарплаты не отображается

### Requirement: Infinite scroll подгрузка
При скролле до конца списка MUST автоматически загружаться следующая страница вакансий. Подгрузка MUST использовать `IntersectionObserver` на sentinel-элементе внизу списка. Запрос MUST отправляться только если `hasMore=true` и нет активной загрузки.

#### Scenario: Скролл до конца — подгрузка следующей страницы
- **WHEN** пользователь прокручивает список до sentinel-элемента и `hasMore=true`
- **THEN** выполняется запрос `GET /api/vacancies?page=1&per_page=30` (с текущими параметрами фильтрации), новые вакансии дополняются к списку

#### Scenario: Подгрузка — все страницы загружены
- **WHEN** вернулось меньше 30 вакансий или `page >= pages`
- **THEN** `hasMore=false`, sentinel-элемент удаляется из DOM, подгрузка прекращается

#### Scenario: Подгрузка — достигнут лимит HH API (2000 результатов)
- **WHEN** `page=66` (30×66=1980) и следующая страница вернула бы超过 лимит
- **THEN** `hasMore=false`, подгрузка прекращается

#### Scenario: Подгрузка — ошибка
- **WHEN** запрос подгрузки завершился ошибкой
- **THEN** отображается сообщение об ошибке в конце списка с возможностью повторить; ранее загруженные вакансии сохраняются

#### Scenario: Новый поиск сбрасывает список
- **WHEN** пользователь применяет фильтры (новый поиск)
- **THEN** список сбрасывается, `page=0`, загружается первая страница

### Requirement: Выбор вакансии из списка
При клике на карточку вакансии MUST загружаться полные детали вакансии и отображаться в правой панели.

#### Scenario: Клик по карточке вакансии
- **WHEN** пользователь нажимает на карточку вакансии в списке
- **THEN** выделенная карточка визуально подсвечивается, в правой панели загружаются и отображаются детали вакансии

#### Scenario: Повторный клик на ту же вакансию
- **WHEN** пользователь повторно нажимает на уже выбранную вакансию
- **THEN** ничего не происходит (повторный запрос не выполняется)

### Requirement: Отображение деталей вакансии в правой панели
В правой панели MUST отображаться полная информация о выбранной вакансии: наименование, работодатель, описание (HTML), ключевые навыки, опыт, локация, формат работы, график, тип занятости, зарплата, дата публикации, ссылка на HH.

#### Scenario: Детали вакансии загружены
- **WHEN** вакансия выбрана и детали загружены
- **THEN** в правой панели отображается полная информация с HTML-описанием

#### Scenario: Вакансия не выбрана
- **WHEN** ни одна вакансия не выбрана
- **THEN** в правой панели отображается текст «Выберите вакансию из списка для просмотра деталей»

#### Scenario: Загрузка деталей
- **WHEN** детали вакансии загружаются
- **THEN** отображается индикатор загрузки

### Requirement: Интеграция с Drawer фильтров — кнопка «Применить»
При нажатии кнопки «Применить» в Drawer фильтров MUST выполняться поиск вакансий с текущими параметрами формы. Drawer MUST закрываться после применения.

#### Scenario: Применение фильтров без пресета
- **WHEN** пользователь заполнил форму (text=python, experience=between1And3) и нажал «Применить»
- **THEN** выполняется `GET /api/vacancies?text=python&experience=between1And3`, Drawer закрывается, результаты отображаются в списке

#### Scenario: Применение фильтров с пресетом
- **WHEN** выбран пресет и пользователь нажал «Применить»
- **THEN** выполняется `GET /api/vacancies?preset_id=<uuid>`, Drawer закрывается, результаты отображаются

#### Scenario: Применение фильтров с пресетом и переопределением
- **WHEN** выбран пресет, изменено поле `text` и нажата «Применить»
- **THEN** выполняется `GET /api/vacancies?preset_id=<uuid>&text=<новое значение>`, результаты отображаются

#### Scenario: Применение пустых фильтров
- **WHEN** форма пуста, пресет не выбран, нажата «Применить»
- **THEN** выполняется `GET /api/vacancies` без параметров, возвращаются все вакансии

### Requirement: Vacancy API сервис в main-fe
В `main-fe` MUST быть создан сервис `vacancyService` с методами: `search(params)`, `get(vacancy_id)`. Сервис MUST использовать `apiRequest()` из `lib/api-client.ts`. Метод `search` MUST принимать параметры фильтрации и пагинации (`page`, `per_page`).

#### Scenario: Вызов поиска с пагинацией
- **WHEN** вызван `vacancyService.search({text: "python", page: 0, per_page: 30})`
- **THEN** выполняется `GET /api/vacancies?text=python&page=0&per_page=30`

#### Scenario: Вызов деталей
- **WHEN** вызван `vacancyService.get("12345")`
- **THEN** выполняется `GET /api/vacancies/12345`

### Requirement: Хук useVacancySearch
В `main-fe` MUST быть создан хук `useVacancySearch`, управляющий состоянием поиска: `vacancies` (накапливаемый список), `selectedId` (выбранная), `vacancy` (детали), `loading`, `error`, `page` (текущая страница), `hasMore` (есть ли ещё страницы). Хук MUST предоставлять методы `search(params)`, `loadMore()` и `select(id)`.

#### Scenario: Инициализация хука
- **WHEN** хук монтируется
- **THEN** автоматически выполняется поиск без параметров (`page=0, per_page=30`)

#### Scenario: Запуск нового поиска
- **WHEN** вызван `search({text: "python"})`
- **THEN** `loading=true`, `vacancies` сбрасывается, `page=0`, выполняется запрос, `vacancies` обновляется, `loading=false`, `hasMore` устанавливается по результату

#### Scenario: Подгрузка следующей страницы
- **WHEN** вызван `loadMore()` и `hasMore=true` и `loading=false`
- **THEN** `page` инкрементируется, выполняется запрос, новые вакансии дополняются к `vacancies`

#### Scenario: Подгрузка невозможна
- **WHEN** вызван `loadMore()` и `hasMore=false`
- **THEN** ничего не происходит

#### Scenario: Выбор вакансии
- **WHEN** вызван `select("12345")`
- **THEN** `selectedId="12345"`, загружается `vacancy` через `vacancyService.get("12345")`
