## Purpose

Определить UI компонент Drawer фильтров в `main-fe` с формой всех не-deprecated параметров поиска вакансий, управлением пресетами и кнопками действий.

## Requirements

### Requirement: Кнопка открытия Drawer фильтров
В левой области workspace MUST быть кнопка «Фильтры» с иконкой фильтра. При нажатии MUST открываться MUI Drawer с `anchor="left"`.

#### Scenario: Нажатие на кнопку фильтров
- **WHEN** пользователь нажимает кнопку «Фильтры»
- **THEN** открывается Drawer слева с формой фильтров

#### Scenario: Закрытие Drawer
- **WHEN** пользователь нажимает за пределами Drawer или кнопку закрытия
- **THEN** Drawer закрывается

### Requirement: Область управления пресетами
В верхней части Drawer MUST быть кнопка-селектор пресетов, аналогичная селектору HH аккаунтов в шапке. При нажатии MUST открываться выпадающее меню со списком сохранённых пресетов.

#### Scenario: Открытие меню пресетов
- **WHEN** пользователь нажимает на селектор пресетов
- **THEN** открывается выпадающее меню со списком пресетов и поисковым input сверху

#### Scenario: Поиск пресетов по имени
- **WHEN** пользователь вводит текст в поисковой input
- **THEN** список фильтруется на backend по регистронезависимому совпадению

#### Scenario: Выбор пресета
- **WHEN** пользователь нажимает на пресет в списке
- **THEN** форма заполняется значениями из выбранного пресета, Drawer остаётся открытым

#### Scenario: Выбор «без пресета»
- **WHEN** пользователь выбирает служебный пункт «Без пресета»
- **THEN** форма очищается, выбранный пресет сбрасывается

### Requirement: Элемент пресета в списке
Каждый элемент списка MUST отображать: наименование пресета, кнопку редактирования (иконка карандаша), кнопку удаления (иконка корзины).

#### Scenario: Редактирование наименования пресета
- **WHEN** пользователь нажимает кнопку редактирования на пресете
- **THEN** открывается модальное окно с предзаполненным input для имени; при сохранении имя обновляется через API

#### Scenario: Удаление пресета
- **WHEN** пользователь нажимает кнопку удаления на пресете
- **THEN** открывается модальное окно подтверждения; при подтверждении пресет удаляется через API

### Requirement: Область фильтров — форма
В центральной части Drawer MUST быть форма с полями всех не-deprecated query-параметров `GET /vacancies`. Поля-справочники MUST быть представлены как Select/MultiSelect с закрытым списком значений из API справочников. Пользователь MUST иметь возможность выбрать только значения, присутствующие в справочнике; свободный ввод текста в поля-справочники MUST быть заблокирован. Поисковая строка MUST присутствовать в select с более чем 5 элементами для фильтрации списка.

#### Scenario: Загрузка справочников при открытии Drawer
- **WHEN** Drawer открывается впервые
- **THEN** загружаются все необходимые справочники (регионы, профессии, отрасли и т.д.) через API

#### Scenario: Поле text
- **WHEN** пользователь вводит текст в поле «Текст поиска»
- **THEN** значение сохраняется в состоянии формы

#### Scenario: Select для experience (закрытый список)
- **WHEN** пользователь открывает select «Опыт»
- **THEN** отображаются только значения из справочника: Без опыта, От 1 до 3 лет, От 3 до 6 лет, Более 6 лет; ввод произвольного текста невозможен

#### Scenario: MultiSelect для area (закрытый список)
- **WHEN** пользователь открывает multiSelect «Регион»
- **THEN** отображаются доступные регионы с поиском по списку; выбранные регионы отображаются как chips; ввод произвольного текста невозможен

#### Scenario: Блокировка свободного ввода в справочных полях
- **WHEN** пользователь пытается ввести текст вручную в select/multiSelect справочного поля
- **THEN** ввод игнорируется; выбор возможен только из выпадающего списка

#### Scenario: Поле salary
- **WHEN** пользователь вводит значение в поле «Зарплата»
- **THEN** принимается только числовое значение

#### Scenario: Группировка полей
- **WHEN** форма отображается в Drawer
- **THEN** поля сгруппированы по категориям: Текст поиска, Местоположение, Профессия и отрасль, Условия работы, Зарплата, Метки и дополнительные

### Requirement: Область кнопок
В нижней части Drawer MUST быть 3 кнопки в 2 ряда: первый ряд — «Сброс» и «Сохранить», второй ряд — «Применить».

#### Scenario: Кнопка «Сброс»
- **WHEN** пользователь нажимает «Сброс»
- **THEN** все поля формы очищаются, выбранный пресет сбрасывается

#### Scenario: Кнопка «Сохранить» с выбранным пресетом
- **WHEN** пресет выбран и пользователь нажимает «Сохранить»
- **THEN** пресет обновляется через PATCH API; при успехе показывается Toast «Пресет «<имя>» успешно сохранён»

#### Scenario: Кнопка «Сохранить» без выбранного пресета
- **WHEN** пресет не выбран и пользователь нажимает «Сохранить»
- **THEN** открывается модальное окно с input для имени; при вводе имени и нажатии «Сохранить» в модалке — создаётся новый пресет через POST API; при успехе Toast

#### Scenario: Кнопка «Применить»
- **WHEN** пользователь нажимает «Применить»
- **THEN** пока не реализует никакой логики (заглушка для task-09)

### Requirement: Отображение кнопок
Кнопки MUST иметь следующие параметры:
- «Сброс»: цвет danger, иконка фильтра слева, текст «Сброс», tooltip «Сбросить фильтры»
- «Сохранить»: цвет secondary, иконка дискеты слева, текст «Сохранить», tooltip «Сохранить фильтры»
- «Применить»: цвет primary, иконка поиска слева, текст «Применить», tooltip «Применить фильтры»

#### Scenario: Tooltip при наведении
- **WHEN** пользователь наводит на любую кнопку
- **THEN** отображается tooltip с описанием действия

### Requirement: API-сервис для фильтров
В `main-fe` MUST быть создан сервис `filterService` с методами: `list(params)`, `get(id)`, `create(data)`, `update(id, data)`, `remove(id)`. Сервис MUST использовать `apiRequest()` из `lib/api-client.ts` и соответствовать паттерну `hhAccountService`.

#### Scenario: Загрузка списка пресетов
- **WHEN** Drawer открывается
- **THEN** вызывается `filterService.list()` и результат передаётся в селектор пресетов

#### Scenario: Создание пресета
- **WHEN** пользователь сохраняет новый пресет
- **THEN** вызывается `filterService.create(data)` с текущими значениями формы
