## Purpose

Определить единый визуальный слой `main-fe` на Material UI и автоматическое применение светлой или тёмной темы согласно настройке операционной системы.

## Requirements

### Requirement: Все страницы используют Material UI
`main-fe` MUST использовать Material UI как единый набор визуальных компонентов для `/login`, защищённой главной страницы, HH OAuth callback и общих loading, empty, error и success состояний; приложение MUST подключать совместимую с Next.js App Router интеграцию MUI и MUST NOT сохранять параллельную самописную библиотеку интерактивных компонентов.

#### Scenario: Отображение пользовательского маршрута
- **WHEN** пользователь открывает любой маршрут `main-fe`
- **THEN** видимые интерактивные элементы и состояния страницы отображаются через согласованные MUI components и theme tokens без зависимости от feature-specific глобальных CSS-компонентов

#### Scenario: Серверный и клиентский рендер
- **WHEN** Next.js выполняет первоначальный рендер и последующую hydration страницы
- **THEN** MUI styles подключаются в поддерживаемом для App Router порядке без hydration error и без потери стилей интерактивных компонентов

### Requirement: Тема автоматически следует настройке операционной системы
`main-fe` MUST предоставлять светлую и тёмную MUI palette, MUST выбирать активную palette по системному `prefers-color-scheme` и MUST обновлять оформление при изменении системной настройки во время открытой страницы.

#### Scenario: Система использует светлую тему
- **WHEN** `prefers-color-scheme` сообщает `light`
- **THEN** приложение отображает все маршруты и состояния со светлой MUI palette и читаемым контрастом текста и controls

#### Scenario: Система использует тёмную тему
- **WHEN** `prefers-color-scheme` сообщает `dark`
- **THEN** приложение отображает все маршруты и состояния с тёмной MUI palette без светлых feature-specific поверхностей, нарушающих согласованное оформление

#### Scenario: Системная тема изменилась
- **WHEN** пользователь меняет системную light/dark настройку при открытом приложении
- **THEN** приложение применяет соответствующую palette без перезагрузки страницы и сохраняет текущее feature-состояние

### Requirement: Системный выбор темы не создаёт клиентские данные
`main-fe` MUST NOT предоставлять ручной переключатель темы и MUST NOT сохранять выбранную palette в cookie, local storage или session storage.

#### Scenario: Повторное открытие приложения
- **WHEN** пользователь повторно открывает `main-fe`
- **THEN** тема снова определяется только текущей системной настройкой без чтения сохранённого пользовательского выбора
